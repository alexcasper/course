Normalisation

 

The major objective of relational database design is to be able to group the attributes into relations that will contribute to minimising the data redundancy. Normalisation can assist greatly with this.

Normalisation is an important procedure as it helps to determine the required relations (tables) and attributes (columns) for each relation. It is therefore a technique for producing a set of suitable relations that support the data requirements of a business. The characteristics of a suitable set of relations include the following: 

  • The minimal number of attributes necessary to support the data requirements of the business.
  • The attributes with a close logical relationship are found in the same relation.
  •  The minimal redundancy with each attribute represented only once with the important exception of attributes that form all or part of the foreign keys.

Normalisation tends to be a multi-step process and the primary purpose is to reduce or control data redundancy. Data anomalies refer to inconsistencies in the data whilst data redundancy refers to having the same data located in different places within the database. The following application of normalisation will ensure that the database will be easier for the user to access and maintain the data, and that minimal storage space is consumed on the storage device, which in turn minimises costs. The unnormalised Form is simply when a relation (table) possesses one or more repeating groups.

 

The steps employed in normalisation.

 

The steps required in normalisation include:

  1. First-Normal Form (1NF)
    This permits the elimination of repeating groups following identification of the Primary Key. This will lead to the intersection of each row and column containing one and only one value.
  2. Second-Normal Form (2NF)
    This is derived from the First-Normal Form (1NF) by eliminating any partial dependencies. The partial dependency arises when the fields (or entries) contained in a record (row) depend on only one portion of the primary key.
  3. Third-Normal Form (3NF)
    The Third-Normal Form (3NF) is derived from the Second-Normal Form (2NF) by eliminating any transitive dependencies.
