 Introducing Data Models

 

In the context of databases, data models are described as integrated collections of concepts for describing and manipulating data, relationships between data, and constraints on the data within a business organization.

They are required because the normal DDL is too low-level to describe the data requirements of a business in a way that would be readily understood by a variety of users.

The data models therefore provide higher-level descriptions of the respective schemas. They should provide the basic concepts and notations that will permit the database designers and end-users to communicate unambiguously, and accurately, their understanding of the data belonging to their business organisation.

In other words, the purpose of the data model is to represent data and to make the data understandable. If this is achieved, then it should be easy to design a database.

Components of a Data Model

The data model can be considered composed of three main components:

 A Structural Part

This consists of sets of rules according to which databases can be constructed.

A Manipulative Part

This defines the types of operations that will be permitted on the data which includes the operations that will be employed for updating or retrieving data from the database, and for also changing the structure of the database.

 A Set of Integrity Constraints

This ensures that the data within the database is accurate.

 

The ANSI-SPARC Three-Level Architecture 

In keeping with the ANSI-SPARC three-level architecture, three related data models can be identified:

  • An external data model which represents each user’s view of the business organization:
    • This is sometimes called the Universe of Discourse (UoD).
  • A conceptual data model which represents the logical view that is DBMS-independent.
  • An internal data model which represents the conceptual schema in such a way that can be understood by the DBMS.

The data models fall into three widely varying categories:

  • Object-based → describe data at the conceptual and external levels.
  • Record-based → describe data at the conceptual and external levels.
  • Physical → describe data at the internal level..

Object-Based Data Models

The object-based data models employ concepts such as entities, attributes and relationships.

This has led to the most common type of object-based data model called the "entity-relationship" (E-R) model.

We will expand on E-R models later in this module. 

Record-Based Data Models

Record-based models consist of a number of fixed-format records.

There are three main types of record-based data models: network, hierarchical and relational.

1. Network Data Model

The network data model is typically represented as a collection of records in which the relationships are represented as sets.

The records become organized as graph structures in which the records are nodes (also known as segments) and the sets are edges in the graphs.

2. Hierarchical  Data Model

The hierarchical data model can be considered to be a “restricted” type of network data model. The data is also represented as a collection of records, and the relationships are represented as sets.

However, the hierarchical model only permits a node to possess one parent.

Physical-Based Data Models

Physical data models describe how data is stored on the computer, or across multiple computer servers.

The information is represented as a collection of record structures, record orderings and access paths.

The two most common physical data models are the unifying model and the frame memory. For now, think of these as bridges allowing the transfer of data from the storage devices (internal) to the conceptual data model (logical view).
