 Introducing Oracle Live SQL

 

Oracle Live SQL is a free to use platform by Oracle. It enables you to create tables and databases as seen for SQLite3.

You can also upload SQL scripts which can be run on the platform in order to populate it with tables containing your data.

It contains a range of tutorials and SQL programs which have been shared by other users.

The Oracle Live SQL platform is running the Oracle Database 19c Enterprise Edition (19.8.0.0.0) in the "backend" and is “powered” by Oracle Application Express (APEX).

Go to Oracle Live SQL to access the platform.

 

 Registering and Accessing Oracle Live SQL

When you access the Oracle Live SQL platform, you should arrive at the "welcome screen" which should be similar to the one below (click the tab).

Oracle Live SQL Welcome Screen

You will need to have a user account in order to use the platform. When you click on “Start Coding Now”, you will be taken to a logon screen.

For first time users, click on “Create Account” and then enter the required details in the form as shown in the tab.

The Interface for Creating an Oracle Account

You should enter your details including your NCH email address.

Once logged in, you should see the following interface (click the tab below).

If not, click again on the Oracle Live SQL link above and then select “Start Coding”.

The Oracle Live SQL Interface

The SQL Worksheet awaits input of SQL code. Once entered, it can be Run from which the output appears in the lower pane.

The tabs on the left side of the interface permit access to the following (Click on the tabs to learn more):

SQL Worksheet

This is where you type in the SQL statements/queries in order to create, update or delete tables in the database.

My Session

  • Previous Sessions:
    • This keeps a record of the log in sessions. Every time you log in to the platform, a new session is created.
    • You can also view what you did in the previous sessions.
  • Utilization:
    • This shows how much resources you have used in your sessions.
    • There are certain limits for example the number of private scripts which can be stored, and the maximum size of a database that can be imported (1MB).
  • NLS:
    • This is the National Language Support and ensures consistency of SQL statement usage across sessions.

Schema

  • This shows the tables currently present during the session.
  • You can also explore the properties of each of the tables such as the attributes and any constraints.

Quick SQL

  • This is a type of “shorthand” that helps in generating the SQL code that will then generate a relational database model.
  • The intention is to save time and effort by automatically creating tables, views and triggers.

My Scripts

This shows a history of scripts that have been created or uploaded. This is useful if you need to regenerate a database after logging in to start a new session.

My Tutorials

This keeps track of any tutorials you have searched and accessed.

Code Library

This holds a mix of tutorials and coding SQL scripts.

You can explore the tabs for yourself in your own time. The two most tabs you will likely use are SQL Worksheet and Schema. 

 

 

 Coding in Oracle Live SQL

The image below shows the creation of the People table created previously in SQLite3 (in the previous lesson).

 

This demonstrates an example of the different use of the SQL keywords by different providers, notably for “auto-incrementing” the IDno attribute.

In SQLite3, we saw that the keyword AUTOINCREMENT was employed, and that we were able to make the IDno attribute a Primary Key immediately on creation of the attribute.

However in Oracle Live SQL, the AUTOINCREMENT keyword is not permitted. Instead we have to use “GENERATED ALWAYS AS IDENTITY (START WITH 1 INCREMENT BY 1)”. We then have to alter the People table to make the IDno attribute a Primary Key.

If you click on the tab below, you can see the schema with the People table shown. There is also a second table displayed which keeps count of the number of entries of the IDno attribute inside the People table, as was seen for SQLite3.

The Schema showing the People table and sequence

The current Schema showing the People table created in Oracle Live SQL along with the “hidden table” which keeps track of the number of entries in the People table due to the “auto-incrementing”.

On completion of the session, you can log out of the platform by clicking on your username and selecting “Sign Out”.

Note: If there is inactivity during the session of more than 20 minutes, the session may time out. If this occurs, you may have to log back in and then select Previous Session from within My Session and select and rerun the previous session. This should also rerun all the commands and queries from that session. Alternatively, you could go to My Scripts and rebuild the database by running the script that was uploaded on the previous or first session. Any commands previously entered will not be rerun.
