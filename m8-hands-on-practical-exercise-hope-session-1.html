 Hands-On Practical Exercise (HOPE): Session 1 

 

Type: Challenge-Based Learning

Description:

A US-based client is considering employing the services of your data analytics department. You have been assigned to carry out the original requests sent over from the prospective client in order for them to assess the competencies of the department. 

The name of the prospective client is Just Lee Books and they have sent over their database via email (was this a good idea?) in two formats:

  • For upload to SQLite3: JLDB_HOPE_1.db  

  • For upload to Oracle Live SQL: JLDB_HOPE_1.sql
    Note: If a new tab opens, then click immediately on "Download JLDB_HOPE_1.sql" - do not wait for the window contents to display.

You should download both versions to your preferred working directory.

Uploading JLDB_HOPE_1.sql to Oracle Live SQL

In order to upload the JLDB_HOPE_1.sql file to Oracle Live SQL, please do the following:

  • Go to 'My Scripts' after logging in, and select "Upload Script".
    • In the pop-up window, you browse to and select the JLDB_HOPE_1.sql file.
    • Enter a name in the 'Script Name' field, e.g. "JLDB_HOPE"
    • Enter a brief description in the 'Description' field.
    • Click 'Upload Script'.
  • After a few seconds, you will see the script displayed in segments on the screen based on SQL statements.
  • Click on 'Run Script'.
    • The database should then be generated for use in Oracle Live SQL.
    • A message should advise that the script ran correctly and you can then proceed to the 'SQL Worksheet'. 

 

Please note that once you have uploaded a database to Oracle Live SQL, you generally do not need to upload it again when you log back in because it should be available in the 'My Scripts' section. When you enter it:

  • Select the database script,
  • Click on "Run Script" and wait for the confirmation message.
  • Once successful you can go to 'SQL Worksheet' to query the database.

Please allow 30 to 60 minutes to complete both stages before you move on to the solutions below.

You will not need to upload your attempts. However, you can choose to save your SQL queries in a separate text file or as screenshots if you want to keep a record of your attempts.

Stage 1

You can use SQLite3 to complete this stage.

The requests from the client are thus:

  • S1-1. List only the contact name and the phone number for each publisher in the Publisher table,
    • Show the column heading for the publisher contacts as “Contact Name” and the number as “Contact Number”.
  • S1-2. List all of the book categories but make sure they only appear once.
  • S1-3. List the ISBN and Title of all books in the “Cooking” category, and another category of your choice .
  • S1-4. List the ISBN, title, price, cost and gross profit (price – cost) for all books .
  • S1-5. Create a mailing list for the customers .
    • The name must appear as would be written on an envelope (i.e. FirstName LastName).
    • The resulting table should have the following headings: Customer Name, Street Address, City, State, Zip Code.
  • S1-6. Provide a list of all the books with a 5% retail price increase .
    • Show the ISBN, title, current and increased price.
  • S1-7. List the author IDs for the stocked books, showing each author ID only once – even if authors have multiple books stocked .
    • Hint: The Books table contains the 'stocked books'.

 

Stage 2

You can use Oracle Live SQL to complete this stage.

Your supervisor has suggested you create a CATEGORY table within the database because there are many books that fall into the same category, and this will maintain data integrity.

First log in to your user account on Oracle Live SQL.

See the above tab ("Uploading JLDB_Build.sql...") for a reminder on how to upload to Oracle Live SQL.

If the database script has already been uploaded, you can go to "My Scripts" and select the script an run it. On successful execution, go to 'SQL Worksheet'.

 

The following bullet points must be achieved successfully if you are to impress both the supervisor and the client:

  • S2-1. Create a new table called Category for the JLDB_HOPE_1 database. It must have the following attributes:
    • Cat_Code: This is a 2-digit auto-incremented number and must be a Primary Key.
    • Cat_Title: This will be 10 characters or less containing brief description like “Computing”. This must not contain NULL values.
    • Cat_Description: This will be a variable length description which will contain things like “Cookbooks, healthy eating and special diet advice” or “Computer and related technology including textbooks and training material”.
  • S2-2. Add all of the categories to the Category table .
  • S2-3. Add a new column for the 2-digit Cat_Code in the Books table .
  • S2-4. UPDATE the new Cat_Code column by using a WHERE clause to SET the Cat_Code value for all of the categories in the Books table.
    • Look at the additional URL for guidance: https://www.w3schools.com/sql/sql_update.asp
  • S2-5. Now that you have the Cat_Code column inside the Books table, you would proceed to delete the Category column from the Books table . 
    • Write the SQL statement you would use in Oracle Live SQL to “drop” the Category column from the Books table.
  • S2-6. Add a Foreign Key constraint between the Category table and the Books table .
  • S2-7. List the relevant data from the new/modified Books and Category tables .
