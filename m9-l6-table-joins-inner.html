 INNER JOIN

 

An INNER JOIN tends to be the most common of joins used.

The following video illustrates an example of INNER JOIN.


Transcript 

INNER JOIN is similar to CROSS JOIN except that the number of rows in the output table are limited by a conditional expression usually stated in the WHERE clause.

This expression is normally used to test the equality between similar columns, for example the AlbumId in both the Track and Album tables.

The outcome of an INNER JOIN leads to an output table in which the rows of the Album table are correctly matched to the corresponding rows in the Track table.

In an INNER JOIN, only matches are returned in the result.

Note that if we intend to perform an "inner join" it is usually not necessary to supply the INNER keyword, and on some occasions JOIN also, because it is implied that the join will be INNER.

 We can apply the same conditions to the Chinook database. For example, we could say:

SELECT Title, TrackId
FROM Album, Track
WHERE Album.Albumid = Track.AlbumId
LIMIT 25;

When you copy this into the console, you should see an output of two columns (“Title” and “TrackId”) with 25 rows.

Think About It!

If you omit the LIMIT 25, what happens? What is the actual size of the output table, i.e. the number of rows? Compare this to the number of rows in the original tables.

Reveal Answer

Reveal Answer

There are 25 rows in the output table because of limiting that output to 25. If the LIMIT 25 is removed, then the output table seems to scroll for a few milliseconds. In order to determine the number of rows, we can count both the Title and TrackId records using COUNT:

SELECT COUNT(Title) AS “Num of Title”, COUNT(TrackId) AS “Num of Track”
FROM Album, Track
WHERE Album.AlbumId = Track.AlbumId;

The values should each be 347. The numbers of rows in the Album and Track tables are:

SELECT COUNT(Title) AS “Num Rows in Album”
FROM Album;

SELECT COUNT(TrackId) AS “Num Rows in Track”

FROM Track;

 

The numbers are respectively 347 and 3503.

 

INNER JOINS: Three Approaches

Here are three other approaches for INNER JOINS which omit the need for the WHERE clause. However, the outputs of all three SQL statements are the same (please click on the 'Three Approaches' tab to learn more).

Three Approaches

Approach

SQL Statement

1

SELECT Title, TrackId
FROM Album NATURAL JOIN Track
LIMIT 25;

2

SELECT a.Title, t.TrackId
FROM Album a JOIN
Track t ON a.AlbumId = t.AlbumId
LIMIT 25;

3

SELECT Title, TrackId
FROM Album JOIN Track USING(AlbumId)
LIMIT 25;

 

 

The outputs of all three SQL statements are the same. See for yourself!!

We can see that the first statement (Approach 1) uses the NATURAL JOIN which identifies that there are two identically labelled columns – one in each table – and will use them to link the two tables and produce the output.

The second SQL statement (Approach 2) has made use of table aliases (a and t) and the ON keyword which looks similar to using WHERE. We will see more use of the ON keyword when dealing with OUTER JOIN commands.

The final SQL statement (Approach 3) makes use of the USING keyword. This appears to be self-explanatory as it “uses” the AlbumId in both tables to link the two tables and meet the conditions.

If there does not exist an attribute (column) with the same name in the second table then use ON.
