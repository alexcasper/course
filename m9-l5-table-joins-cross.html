 Cross Joins

 

Earlier in the module, you saw how using subqueries can help fine tune a query in order to hone in on a particular piece of information. However, this may still not be sufficient to achieve the overall goal.

It is usually therefore necessary to join tables together in order to retrieve the information, and thus gain additional insights.

Performing joins are therefore the most important and powerful database operation which permits data stored on different tables to be brought together.

When we look at a single table, we are only getting a limited piece of information. This will be insufficient in answering business-related questions that require adequate answers and significant insights.

This is also in conjunction with some tables containing potentially hundreds, thousands and even millions of rows – the effort required to manually match up the information in order to retrieve the insights required would be substantial.

To overcome these issues, SQL can be employed to temporarily join two or more tables into one (large) table, which thus contains all of the required information in one location and permit targeted information retrieval.

How these tables are joined together will depend on the actual relationship between the tables, and the information that is to be retrieved. 

There are therefore a number of different approaches to joining tables together:

  • Cross Join
  • Inner Join
  • Left Outer Join
  • Right Outer Join
  • Full Outer Join

These approaches will be explained in the following lessons. In this lesson, we will look at Cross Joins.

The outcome of a CROSS JOIN is simply to match each row from the first table to all of the rows in the second table.

The figure below illustrates the CROSS JOIN approach.

The CROSS JOIN approach will result in very large tables being generated. Therefore they should be used with extreme caution. It is possible to calculate the size of the product table from a CROSS JOIN as:

  • The number of rows will be the product of the number of rows of the two tables.
  • The number of columns will be the sum of the number of columns in both tables.

If Table 1 contained a list of colours (Red, Blue and Yellow) whilst Table 2 contained a list of sizes (Large, Medium and Small), what would be the outcome of the CROSS JOIN?

Think About It

If the first table contains 30 rows and the second table contains 50 rows, then what will be the final number of rows when the two tables are joined?

Reveal Answer

The number of rows will be the product of 30 and 50, i.e. 30 × 50 = 1500.
