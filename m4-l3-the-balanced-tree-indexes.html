The Balanced Tree Indexes

 

The balanced tree index (also known as B-tree ) is commonly employed in database systems using Oracle. The ‘basic’ SQL statement CREATE INDEX  will generate a B-tree  index. For example, if a zip code in the ‘Just Lee bookstore ’ database is being searched, the index can help locate the position(s) far more rapidly. Before we can take advantage of this we need to create the appropriate index.

Please be sure to click on both of the tabs below:

CREATE INDEX

CREATE INDEX customers_zip_idx
ON Customers (Zip);

As you can see the name of the index refers to the table, then the attribute, then the _idx ending to represent an index (just like _seq represented a sequence). In reality, you can call the index anything you like, but it makes sense to give it a name that can help identify what table it belongs to and which attribute, and the fact that it is indeed an index.

Creating Index in SQLite3

Creating an index is permissible in SQLite3, and so the remainder of any examples will be shown with this. Therefore start up the Command Prompt window, change the directory to the location of the SQLite3.exe file, and then start the application, e.g. sqlite3 jldb.db. 

Remember to state .tables to check that the database has loaded correctly. Then type in at the prompt .header on and .mode column. After you have created the index shown above, you can type .indexes or .indexes customers and you should see the indexes that are current in the loaded database as shown in the figure below. 

 

Viewing the indexes present in the loaded database in SQLite3 and focusing on the Customers table. Observe the newly created index: customers_zip_idx.
