 OUTER JOIN

 
As seen with the INNER JOIN, if one row of a joined table is unmatched, then that row is omitted from the output table.

With OUTER JOIN operations, the rows that do not satisfy the join condition are retained. The remainder of that row in the output table will be padded out with NULL values.

There are three approaches in implementing OUTER JOIN operations.

LEFT (OUTER) JOIN

RIGHT (OUTER) JOIN

FULL (OUTER) JOIN

The unmatched rows tend to be included from the first table.

The unmatched rows tend to be included from the second table.

The unmatched rows tend to be included from both of the tables.

Joins are thus incredibly useful and powerful approaches in relational databases. They will enable a more thorough extraction of information leading to better analytical results and insights needed for successful business outcomes.

Whilst the order of the tables does not matter if performing an INNER JOIN, the order of the tables in an OUTER JOIN does matter.

For example, whether Table 1 is on the “left” or on the “right”, an INNER JOIN produces the same output table; that for an OUTER JOIN will however produce different output tables.

This means that for LEFT or RIGHT joins, the order of the tables does matter because it states which table the unmatched results are returned for.

The “left” table (e.g. "Table 1") is actually specified using FROM whilst the “right” table (e.g. "Table 2") is specified using LEFT JOIN or RIGHT JOIN.
