 SELECT and FROM

 

In the next few lessons, we will take a look at some of the common keywords employed in SQL such as SELECT, FROM, WHERE, GROUP BY, HAVING and ORDER BY.

This is the actual order of the SQL keywords and clauses and cannot be changed.

SELECT [DISTINCT | UNIQUE] (*, columnName [AS alias], ...)

FROM tableName

[WHERE condition]

[GROUP BY group_by_expression]

[HAVING group_condition]

[ORDER BY columnName];

The only two keywords that are mandatory are SELECT and FROM. The other keywords and clauses (e.g. DISTINCT) are optional (hence encased in square brackets).

The SELECT and FROM SQL keywords are the ‘backbone’ of SQL programming as together they permit retrieving the required information from tables (relations) stored within the database.

  • SELECT: Identifies the columns required from one or many tables.
  • FROM: Identifies the table or tables containing the information (data).

Queries and statements can be ‘very simple’ such as “SELECT * FROM table_name;” to very complex queries which may contain mathematical and/or statistical calculations, extracting information from multiple tables within a database, and “nested” SQL queries, in other words SQL queries within SQL queries (i.e. a subquery).

The combination of the SELECT and FROM keywords therefore retrieve the data from the database. The syntax provides the basic rules for a command. We will discuss other keywords such as WHERE and GROUP BY later in the course.
