  Summary & Looking Ahead

 

In this module, you have seen how to limit outputs (the number of records to be shown in the console) as well as how to offset the output. You have seen how equality, relational and logical operators are employed, and in what order certain expressions are evaluated, and that parentheses can change the order of the evaluation.  

You have been reminded about the usefulness of the GROUP BY and HAVING clauses, leading to the usefulness of subqueries (sometimes nested!) to better fine tune your SQL statement/query. You have also seen how MERGE statements with conditions can be employed for either updating or inserting records into a given table from other tables. You have seen how tables can be joined together via columns and/or rows in order to fine tune your SQL query and thus extract the required item(s) of information from the database.

You have seen that there are inner, outer and self joins for the columns. For the rows, you saw how the use of sets can generate the combined tables; these sets simply link SQL queries together, sometimes in excess of two, to produce an overall output table.

Next, you will take a closer look at data objects such as views, sequences and indexes, and functions, notably single-row and multiple-row functions that can perform specific actions whilst a SQL statement is being executed.

 

Learning Objectives

 You should now be able to:

  • Understand the basic principles of relational databases: their structure, use and how to interrogate them. Basic SQL syntax and query structure. Design and implement basic relational databases.
    • Explain additional key functions and clauses in SQL for processing tables in a relational database.
  • Code structurally correct SQL commands. Perform basic data manipulation and information retrieval operations using SQL. Demonstrate skill in numeracy and computation. Understand logical arguments, identifying the assumptions made and the conclusions drawn. Demonstrate the ability to analyse, evaluate and correctly interpret data.
  • Describe procedures for filtering and sorting data.
  • Explain methods of joining data from multiple tables.
  • Describe Sets.
  • Perform coding in SQL.

If you are not able to confidently discuss the above, you should revisit the material and perform self-directed research.
