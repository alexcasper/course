Logical Operators and NULL

 

Logical operators involve the following: AND, OR, NOT, BETWEEN, IN and LIKE. You have already been introduced to some of these in Module 2. You can revise the types of logical operators.

Please be sure to click on all of the expanding tabs shown on this page.

Logical Operators: Types 

AND:

This permits the existence of multiple conditions when using the WHERE clause.

OR:

This is used to combine multiple conditions when using the WHERE clause.

NOT:

This reverses (or ‘negates’) the meaning of the logical operator.

  • If TRUE OR FALSE TRUE then NOT makes it FALSE.
  • If TRUE AND FALSE FALSE then NOT makes it TRUE.

BETWEEN:

Values are searched in which they sit within a specified range (the minimum and maximum value).

IN:

This permits comparing a value with a list of literal values. If there is a positive match (TRUE) then the value is chosen. It is like having “multiple ORs”.

LIKE:

This permits the comparison of a value with similar values using wildcard operators.

 

The BETWEEN Clause

The BETWEEN operator can be used in place of relational operators and also other data types.

BETWEEN Operator Instead of Relational Operator 

The BETWEEN operator can be used in place of relational operators as shown below.

SELECT CustomerID, InvoiceID, Total
FROM Invoice
WHERE Total BETWEEN 10 AND 13;

Multiple Logical Operators

A final point to note is when multiple logical operators are used. Recall that multiplication and division are evaluated first before addition and subtraction, the same also occurs with logical operators.

The following video will demonstrate how the parentheses affects the order of the logical operators.

Order of Evaluation

In fact, because the WHERE clause can possess multiple types of operators, you need to be aware of the full order in which all of the operators combined are resolved:

  • Arithmetic Operators:
    • Multiplication and Division
    • Addition and Subtraction
  • Relational Operators:
    • In no particular order: <, >, =, LIKE.
  • Logical Operators:
    • These tend to be evaluated last and are done so in the order: NOT, AND, and OR.

If you need to change the order of the evaluation, then you make use of parentheses in order to define which operators get resolved first. View the video for an example that illustrates this well including the importance of parentheses.

NULL

We have seen in Module 2 when NOT NULL is used to specify during table creation that entries into attributes are not ‘blank’, in other words not empty. This is especially the case for attributes that are designated as primary keys.

We can also use NULL in combination with IS to check for conditions when using the WHERE clause.
