 Entity-Relationship Model

 

An entity-relationship (E-R) model is an object-oriented model and can be used to describe the relational structures in a database.

The E-R model is normally presented as using entity relationship diagrams (ERDs).

ERDs are very useful in designing a database. They are also very useful in describing existing databases, especially the very large ones in which there is an important requirement to understand how different tables within the database are related and work together.

The ERDs employ simple boxes, lines and symbols in order to illustrate the entities, attributes and relationships between the data.

Entities

Entities comprise unique identities such as people, things or concepts in which the information they convey is very important to the business (e.g. a product, customer, supplier, property or order).

They can exist as physical or conceptual:

  • Physical existence includes “staff”, “property”, “customer”, “part”, “supplier” and “product”.
  • Conceptual existence includes “viewing”, “inspection”, “sale” and “work experience".

Attributes

Attributes, as can be seen, are the properties pertaining to the characteristics of an entity such as a name, type or colour.

Relationships

The relationships describe how the data is shared between the entities by the lines joining (or linking) the two tables.

We can also observe how the entities (tables) are linked using the primary and foreign keys.

In the ERD, these are indicated by the lines shown between the entities, and also describe how the entities are linked using symbols defined by the cardinality and the modality (see next tab).

Cardinality and Modality

The cardinality and the modality are described by the styling of the line and what the endpoint looks like.

This also depends on the chosen notation style. They both concern ‘business rules’ around a relationship between entities. 

  • Cardinality refers to the maximum number of times an instance in one entity can be associated with instances in the related entity. 
  • Modality refers to the minimum number of times an instance in one entity can be associated with an instance in another entity.

The notations can be inserted either side of the relationship (denoted by the line) and therefore permits the full comprehension of the relationship between the two tables (entities).

Common Types of Relationships

There are three common types of relationships in ERDs.

 One-to-one

  • This is the most basic kind of inter-table relationship. This is when a relationship is established between one row (tuple) of a table with one row from another table.

One-to-many

  • This is when a relationship is established between one row of a table and many rows of another table.

Many-to-many

  • This can be seen as two one-to-many relationships constructed over each other.
  • This permits one row from the first table to be linked to many rows in the second table whilst at the same time one row from the second table is linked to many rows in the first table.

Reveal Answer

 A relation has the following properties:

  • A name that is distinct from all other relation names in the relational schema.
  • Each cell contains exactly one atomic (single) value.
  • Each attribute has a distinct name.
  • The values of an attribute are all the same domain (i.e. data type).
  • Each tuple (row) is distinct which means there are no duplicate tuples.
  • The order of the attributes has no significance.
  • The order of the tuples should have no significance but this may depend on its position in the table if there are thousands of rows present.
