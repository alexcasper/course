 User Interfaces

A user interface is fundamental to the workings of an information system. The first digital computers made use of physical methods such as punch cards and printers to input instructions and output results. A terminal, allowing for text to be entered and displayed, was an advance over this. Command-line applications are still used for many more complex applications with a limited scope. To make systems more friendly, however, graphical user interfaces were developed, allowing for easier navigation through computer operating systems. Desktop applications could also be downloaded, with interfaces catering to their specific functionality. The increased availability of smartphones has led to the design of mobile applications, with interfaces catered towards smaller touchscreens. More recently, the development of cloud-based software-as-a-service platforms and web applications has meant many systems are now accessible using only a web browser. Experiments with virtual reality environments, voice recognition and text-based chatbots continue; each of these provide an ability to interact with information in a different way.

Changing between Polaris and UI16

Note that this is a slightly complicated exercise and should not be attempted until you are somewhat comfortable with the platform. Being able to explore and compare earlier interfaces however is an excellent exercise for building your knowledge of a platform. It also might be necessary if you want to follow an older tutorial or book. Doing this is not necessary for course completion.

In order to set a San Diego instance to use the older user interface, select the All button in the top left. Type sys_properties.list to bring up the list of all system properties.

 

In the search bar at the top, select Name from the drop town and type *polaris. Hit enter.

You should see a property called glide.ui.polaris.experience. If you change this to false, it will put you back into the older interface, called UI16. Note: If this doesn't work, try deleting everything after the url and hitting enter, i.e. https://dev11111.service-now.com/



To put yourself back into the new interface Next Experience/Polaris, you can simply go to the same sys_properties.list via the UI16 application navigator.

Changing between UI16 and UI15

If you want to go even further back in time, switch to UI16 using the steps above, and then click the cog in the top right.

On the system settings, select the 'Switch to UI15' button in the top right.

To get back from UI15 to UI16 is a similar process. Click the cog in the top right, then switch to UI16.

Note there is no direct way to switch between Polaris and UI15 - you need to go via the intermediate interface.
