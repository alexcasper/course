Tables

Underlying ServiceNow is a developed relational database, with thousands of tables. The efficiency of the platform draws heavily from the core efficiency of relational databases. While relations between tables are key to the operations of ServiceNow, this platform has some functional components that further expand the database's usability. An example of this would be that within an instance, there is a unique identifier for each record regardless of which table it is in. The existing data organisation - called a schema - helps organisations adopt best practices without the risk or expense of creating their own database.

UI Elements

The User Interface elements are what differentiate ServiceNow from a more traditional database system. The interface makes the data easily accessible and modifiable without knowledge of Structured Query Language (SQL).

Application Files

Application Files in ServiceNow are the mechanisms by which data is transformed within the platform itself, or as it is being transferred into the platform. Concepts such as Business Rules, Workflows, Flows and Scripts are elements which facilitate the mechanics of the platform.

Integrations

Integrations are the mechanisms by which data is interchanged with other platforms. The ServiceNow platform has a number of different capabilities allowing it to exchange data with other Platforms and Application Programming Interfaces (APIs).

Dependencies

Dependencies includes all the other tables and applications an application requires to operate. These can include both out-of-the-box tables and applications - which are included with the default installation of ServiceNow - and other custom applications.
