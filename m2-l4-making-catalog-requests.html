Request Capture

Allowing end users to request services through a portal allows us to capture information about the resources being requested, allowing for more effective fulfilment of those requests. In addition, it allows us to later perform analysis of the types of requests being made, as well as how quickly or successfully they are then met.

Submission of service requests within ServiceNow is typically done through the Service Portal, but can also occur through the internal-facing interface, through a form, or through another application through the use of the ServiceNow API.

Making a Request from Service Portal

As mentioned on the previous page, there are other ways to directly input data into the request management system. We can, for example, as an admin access the tables directly and enter records. At the same time, in this case it breaks the request approval flow - as the request needs to be created before the requested items, the request with no items is automatically approved. Therefore we would need to design a flow or script to take account of this.
