 Single-Row Functions

 

Functions are predefined blocks of code that accept input arguments, and can output a single output or multiple outputs.

You will have possibly seen and used functions in the “Intensive Foundations in Computer Science and Programming I” course. 

In SQL, we can have:

  • Single-row functions: These return one row of results for every record that is processed.
  • Multiple-row functions: These return one result per group of data that is processed. 

Let's take a closer look at single-row functions.

The table below shows some of the typical single-row functions available in SQL and the type of function they perform.

Case Conversion Functions

The case conversion functions permit the alterations of the cases of letters stored in an attribute (column) or a character string.

For example, all the letters in a person’s surname can be transformed into upper case using UPPER or into lower case using LOWER.

This can be useful when performing a case-sensitive comparison. For example “John Smith” is not the same as “jOhn SmitH”, however if we convert this to lower case, i.e. “john smith” then we stand a good chance of a ‘positive hit’ if we were searching for a customer called “John Smith”. 

The case conversion functions can be used in the following situations: 

A SELECT Clause

A WHERE Clause

They can alter the appearance of the data in the results.

 

They can alter the value for comparison.

 

 

The INITCAP function converts all of the first letters of the collection of words, e.g. a person’s name, into capital letters.
