Summary & Congratulations

In our final week, we explored how the popularity of Cloud Computing has lead to organisational transformations. In particular, we will looked at how the Cloud supports Agile delivery of software, services and Data insights. We covered DevOps, the practice of reconciling the previously separate practices of system development and operations, and the new field of Solution Architecture. We also looked at the way in which Cloud facilitates Ubiquitous Computing and the so-called Internet of Things. The final lab, which was not part of the Summative Assignment, looked at analysing IoT data using AWS IoT Core and IoT Analytics.

Congratulations! You have completed all learning materials for Cloud Computing.

Readings

Ch. 5. Cloud Computing. In: Stawski, S. 2016. Inflection Point: How the Convergence of Cloud, Mobility, Apps and Data Will Shape the Future of Business. Saddle River: Pearson. via O'Reilly.

Apprenticeship Standard

This week’s learning objectives align to the following apprenticeship standards.

Data Scientist

  • K1: The context of Data Science and the Data Science community in relation to computer science, statistics and software engineering. How differing schools of thought in these disciplines have driven new approaches to data systems.
  • K2: How Data Science operates within the context of data governance, data security, and communications. How Data Science can be applied to improve an organisation’s processes, operations and outputs. How data and analysis may exhibit biases and prejudice.  How ethics and compliance affect Data Science work, and the impact of international regulations (including the General Data Protection Regulation.)
  • K3: How data can be used systematically, through an awareness of key platforms for data and analysis in an organisation, including:
    • Data processing and storage, including on-premise and cloud technologies. 
    • Database systems including relational, data warehousing & online analytical processing, “NoSQL” and real-time approaches; the pros and cons of each approach.

Skills

  • S2: Perform data engineering: create and handle datasets for analysis. Use tools and techniques to source, access, explore, profile, pipeline, combine, transform and store data, and apply governance (quality control, security, privacy) to data.
  • S5: Implement data solutions, using relevant software engineering architectures and design patterns. Evaluate Cloud vs. on-premise deployment.  Determine the implicit and explicit value of data. Assess value for money and Return on Investment.  Scale a system up/out. Evaluate emerging trends and new approaches. Compare the pros and cons of software applications and techniques.

Behaviours

  • B4: Consideration of problems in the context of organisation goals.

Learning Objectives

You should now be confident with the following learning objectives:

  • Understand the key principles and underlying basic concepts of cloud computing, cloud services, cloud data warehousing and cloud storage applied to data science
    • Explain how computing supports agility in data science and computing in general
    • Describe the roles of Dev Ops Engineer and Solutions Architect
  • Have knowledge of how international regulations, ethics and compliance affect cloud computing services and the implications to the user
    • Discuss key challenges and operational considerations for cloud computing
  • Use cloud technologies for data processing and storage
    • Demonstrate how AWS IoT Analytics can be used to analyse streaming data from internet of things devices.
  • Analyse, evaluate and correctly interpret data
    • Create effective queries based on streaming data from IoT devices

If you are not able to confidently discuss the above, you should revisit the material and perform self-directed research. If you are still having difficulties, please post a question on the discussion forum.
