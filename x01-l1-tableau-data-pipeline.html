Tableau Data Pipeline

Before we start using Tableau, let's explore some basic data analysis methods. The first concept to introduce today is the data analysis pipeline. 

What is a data analysis pipeline?

A data pipeline is a set of steps that we need to follow to ingest raw data from various sources and move the data to a destination, in our case Tableau for storage and analysis. During these steps, we will be able to work with data, including data filtering and exploring relationships.

The data pipeline could include simple processes including data cleaning, extraction and loading, or it may be designed to handle data in a more advanced manner, such as modelling and using machine learning for data analysis. In our case, we will focus on the fundamental aspects of data management using Tableau, including the following steps:

Using various data sources

Data sources could include relational databases, raw datasets or online data that is under an application programming interface. In this module, we will focus on raw data sources, and we will explore the use of comma-separated values (CSV) files.

Data storage for analysis

There is a variety of data storage options, including cloud services, data stores or databases. In our case, we will focus on the use of Tableau to store data, so we will simply load files (e.g. by dragging and dropping files).

Data cleaning and transformation

This is a key step, prior to any data analysis step. The transformation refers to operations that change data, which may include data standardization, sorting, deduplication, validation, and verification. The ultimate goal is to prepare the dataset for smooth data analysis.

Data exploration

This step is the first effort to visualise and examine the dataset for possible correlations or identification of patterns in the data. In many cases, data observation can also help in the data cleansing phase. The observation phase usually refers to plotting data to visually identify relationships for the next pipeline step.

Data querying

At this point, we can start querying the data to explore the hidden knowledge. This includes queries to select and filter data according to different criteria.

Data visualisations

The data visualisation can help us to conclude the analysis phases and export meaningful visualisations such as bar, line and geographic graphs.

 

Why learn Tableau?

Let's explore the following advantages.

  1. Tableau is a data visualization tool first and foremost. Therefore, its technology is there to support complex computations, data blending and dashboarding for the purpose of creating beautiful visualizations that deliver insights that cannot easily be derived from staring at a spreadsheet.
  2. Tableau is ideal for creating interactive visual within minutes. The interface can handle endless variations while also limiting you from creating charts that are against data visualization best practices.
  3. There are many different types of visualization options available in Tableau which enhance the user experience. Also, Tableau is very easy to learn compared to Python or R, anyone without having knowledge of coding can easily learn Tableau.
  4. Tableau can handle millions of rows of data with ease. Different types of visualization can be created with a large amount of data without impacting the performance of the dashboards.
  5. To avoid performance issues and to do complex table calculations in Tableau, users can incorporate  Python or R. Using Python script can take the load off the software by performing data cleansing tasks with packages. However, Python is not a native scripting language accepting by Tableau.
