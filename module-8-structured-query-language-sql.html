 Overview

 

You have been introduced to the concepts of relational databases from their design to their importance in business organisations, and the importance of a database management system (DBMS). Regardless of the size of a database (e.g. whether stored on one personal computer, or a mainframe), the ability to efficiently and effortlessly retrieve, insert, delete and update items within a database is of vital importance.

One commonly employed approach is to use SQL statements to interact with the database.

Structured Query Language (SQL) is the most widely employed database query language, and has been for the last few decades. It has been supported by many relational DBMSs (RDBMSs) such as Oracle, Microsoft SQL Server, IBM’s DB2, MySQL and Microsoft Access.

SQL has been a standard for both the International Organisation of Standards (ISO) and the American National Standards Institute (ANSI). SQL tends to be pronounced as “S Q L” or as “Sequel” – the two are interchangeable.

Although SQL is a recognised standard, some of the database providers tend to possess their own SQL editors to which there may be small variations in the syntax which could provide more functionality than the normal standard. This will be dependent upon how the SQL features are implemented by the database providers.

 

Learning Objectives

By the end of this module, you should be able to:

  • Understand the basic principles of relational databases: their structure, use and how to interrogate them. Design and implement basic relational databases.
    • Explain the importance of SQL for accessing, processing and updating the contents of a relational database.
  • Basic SQL syntax and query structure. Demonstrate the ability to analyse, evaluate and correctly interpret data.
    • Detail the definitions, manipulations, functions, keys and clauses in SQL with illustrative examples.
  • Code structurally correct SQL commands. Perform basic data manipulation and information retrieval operations using SQL.
    • Perform coding in SQL.
