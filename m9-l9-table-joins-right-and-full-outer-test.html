Self-Joins

A final join to introduce is the "Self-Join". This is normally employed to link a table to itself.

This requires the use of table aliases along with the use of column qualifiers.

The join can occur via the WHERE clause or via the FROM clause in conjunction with the ON clause.

In the figure below we see a portion of a table (Customer2) and how the Referred attribute indicates which customer is linked to other customers via their CustomerId value.

The SQL statements that follow utilize the the WHERE and FROM/ON clauses which produce the same output.

The self-join simply occurs when a different alias is used for the same table, in this case, c and r for the customer2 table.

This has given the impression that two tables are being joined together when in fact it is the same table.

One final point to note here is that the USING clause cannot be employed with a self-join since two different columns are being used in the join.
