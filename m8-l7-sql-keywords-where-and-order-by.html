The WHERE Keyword

The WHERE keyword is very useful for filtering the rows (records) in a table.

It helps to hone in on the specific piece of information required.

The WHERE keyword can be combined with other keywords such as IN and LIKE as well as logic operators using AND and OR.

In this video, we continue to look at the ‘Just Lee’ bookstore database with SQLite3.




Transcript 

As shown in the video, pattern matching can also be performed when WHERE is being employed.

For example, if you were interested in the orders that had “Ave” or “Avenue” in the ShipStreet attribute in the Orders table (as shown in the video and the figure below), you would use the LIKE keyword in combination with two possible pattern matching symbols (please click on both tabs below for examples of usage):

% (the "percent" symbol)

  • This permits any sequence of 0 or more characters before and after the '%' symbol.
    • Example: With " %and% ” the '%' symbols can be replaced with any number of letters or words such as “Laurel and Hardy”, “ candy ”, “and”, “wonderland”, “android”, etc.

_ (the "underscore" symbol)

  • Any single character can replace the 'underscore' symbol.
    • Example: With “ _and ” the '_' can be replaced with letters to produce the following:  “ band ”, “ hand ”, “ land ”, “ sand ” and “ wand ”.

The ORDER BY Clause

The ORDER BY clause is very useful if you were interested in displaying the output in ascending or descending order, for example from A to Z or “heaviest” to “lightest”, via a specific attribute (or sequential set of attributes)..

The ORDER BY is usually the last keyword to employ when executing a SQL statement (or query).

The example below illustrates three queries:

  • the first without using ORDER BY;
  • the second using ORDER BY to arrange in ascending order (lowest to highest) via the CustomerNo attribute;
  • the third in descending order via the CustomerNo attribute as denoted by the DESC clause.
