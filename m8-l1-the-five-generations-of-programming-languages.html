 The Five Generations: 1GL to 5GL

 

Structured Query Language (SQL) is said to be a “4th generation language” (4GL).

It is considered to be a “shorthand programming language”. This means that one line of code in 4GL could involve, for example, 37 lines of code in a 3GL (“3rd generation language”).

There are currently five generations in total (1GL to 5GL). Please watch the video below for more information.

1GL – First Generation Language

This is classed as a “low-level” programming language as it communicates directly with the hardware in the form of binary numbers  machine language.

2GL – Second Generation Language

This is also classed as a “low-level” programming language in the form of Assembly Code which can be read by humans, but can still be very difficult to understand and follow for humans.

3GL – Third Generation Language

These are “higher-level” languages that are most importantly “machine-independent”. These can be easily read by humans, although a considerable amount of knowledge is still required to be able to construct/write the programs in the respective languages.

3GL contains some of the following common and popular languages: BASIC, C, Fortran, Pascal, Java, R, Matlab and Python.

4GL – Fourth Generation Language

These are also “high-level” languages and are much closer to “human language”, for example, making use of ‘common English words’. In SQL, you will see common words like SELECT, FROM, and WHERE very much used!

The 4GLs are more focused on “what” needs doing rather than “how” to do it.

5GL – Fifth Generation Language

This is deemed to be the “natural language”. The computer is expected to solve the given problem, and not be told how to solve the problem. This is due to making use of “constraints” and/or “logic”.

Examples include Prolog, Mercury and OPS5.

 

In terms of ‘popularity’, 3GL and 4GL are more employed today. Indeed, the majority of data science, database and software developer roles require the applicant to possess extensive knowledge and experience with the likes of Python, R, SQL and sometimes C++ (and/or Java).

5GL does not appear to be as ‘popular’ today because when it came to formulating an efficient algorithm in order to solve a particular difficult problem, the application of the constraints became a harder problem to overcome for very large ‘constraint-based’ programs.
