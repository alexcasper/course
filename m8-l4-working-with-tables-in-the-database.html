 Working with Tables in the Database

 

Tables can be created, populated, updated, emptied and finally deleted.

Click on the tabs below to learn more.

Creating a table

When we create a table, we define the attributes and their domain (datatype):

CREATE TABLE table_name (

Column_1 datatype,

Column_2 datatype,

…

);

When we create a table, we can also specify additional parameters for the attributes (columns) such as NOT NULL, PRIMARY KEY and FOREIGN KEY to give these attributes specific behaviours. In addition to creating and populating tables, you can also delete tables.

Inserting values into a table (populating)

When we insert the values into the table, we can do so depending on whether we have knowledge on all of the attributes present within a table.

If we know all of the attributes and their domains, we can say:

INSERT INTO table_name

VALUES (Value_1, Value_2, Value_3, Value_4, …);

If we only need to populate some of the attributes, we can say: 

INSERT INTO table_name (Column_1, Column_4, Column_7, …)

VALUES (Value_1, Value_2, Value_3, …);

As can be seen, Value_1 is inserted into Column_1, Value_2 is inserted into Column_4 and Value_3 is inserted into Column_7.

It is very important that the datatypes of the values to be inserted into the table match the datatypes of the attributes. For example, a person’s surname (datatype is ‘TEXT’) cannot be inserted into an attribute that has a ‘numeric datatype’.

Updating values inside a table

If you wanted to update a value in a record (row), you can use the following SQL commands:

UPDATE table_name

SET attribute1 = value1

WHERE attribute2 = value2;

In this instance, attribute1 is the column that contains the cell(s) to be updated with value1 provided that the predicate in the WHERE clause is evaluated to be TRUE, which in this case value2 is present in attribute2. 

Deleting values from a table ("emptying")

The DELETE statement can be employed to remove records (rows) from a table. For example:

DELETE FROM table_name;

This deletes all of the records inside a table. This means that the table becomes empty but it still contains the attributes within. Alternatively:

DELETE FROM table_name

WHERE attribute2 = value2;

This deletes the record provided that the predicate in the WHERE clause is evaluated to be TRUE, which in this case value2 is present in attribute2. 

Deletion of actual table ("dropping")

In order to delete (i.e. remove) an entire table from a database, use the DROP statement:

DROP TABLE table_name;

The video gives an important warning about dropping tables in that you must be very sure it is safe to do so, especially if the table is linked with other tables within the database.

For example, you do not want to delete a table that was being used as a “bridge” to link two or more tables together (“many-to-many relationships”).

The video also gave a good example of the specification of a column designated as a Primary Key and that it must not contain any NULL values. It must also auto-increment each time a new record is inserted.
