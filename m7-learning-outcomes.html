  Summary & Looking Ahead

 

You should now understand what a database is and what it consists of along with what it can be used for.

You have also been introduced to the database management system (DBMS) and the key role played by it in accessing and processing the data stored in a database.

You have also been presented with the types of architectures and schemas employed to define a database and how there are typically three levels: external (user), conceptual (logical) and internal (physical). You have seen the importance of schemas in helping to design a database taking into account the conceptual, logical and physical considerations.

You have been introduced to the security requirements of both the database and the computer system, along with the ethics and regulations applicable to IT systems and databases.

In the next module, you will be introduced to programming using SQL (Structured Query Language). You will use SQL to insert, update or delete data from a table within a relational database. You will also apply your knowledge to a small challenge so that your SQL coding skills can be enhanced.

 

Learning Objectives

You should now be able to: 

  • Understand the basic principles of relational databases: their structure, use and how to interrogate them.
    • Explain database in relation to its design and its use in the storage of different types of data.
    • Explain how to manage a relational database.
    • Identify the different environments of a relational database.
  • Have knowledge of the regulations, ethical and security considerations required in modern data storage.
    • Detail the ethical considerations concerning the processing of data within a relational database.
    • Be mindful of the security aspects required to keep information safe on a database.

If you are not able to confidently discuss the above you should revisit the material and perform self-directed research.
