 Overview 

 

Previously, you were introduced to the concepts of Table Operations in SQL. You learnt about the key functions and clauses in SQL for processing tables in a relational database. You also learnt how to filter and sort data and join data from multiple tables. 

Database objects can be identified as containing a name and a defined structure. These include tables, sequences and indexes. 

We have already covered tables elsewhere, and will continue to do so throughout the remainder of the course. As you have seen, tables store the data. 

We will now also cover other important database objects namely sequences, indexes and views, which can be used alongside tables.

 

Learning Objectives

By the end of this module, you should be able to:

  • Demonstrate skill in numeracy and computation.
  • Understand the basic principles of relational databases: their structure, use and how to interrogate them. Understand logical arguments, identifying the assumptions made and the conclusions drawn. Demonstrate the ability to analyse, evaluate and correctly interpret data.
    • Explain sequences, indexes, views, selected row functions and additional aggregate functions.
    • Explain the purpose of UNIQUE and CHECK.
  • Basic SQL syntax and query structure. Code structurally correct SQL commands. Design and implement basic relational databases.
    • Perform coding in SQL.
