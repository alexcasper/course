 Hands-On Practical Exercise (HOPE): Session 3

 

 Type: Challenge-Based Learning  

Description:

The current US-based client your supervisor has assigned you to (“Just Lee Books”) have requested more in depth information that they would like extracted from their database.

Pleased with your contribution to the internal project from last week, your supervisor has expressed confidence to the client that you will come through for them.

To help you, the supervisor has suggested you tackle the tasks in two stages: using row functions and then indexes.

You have also been advised that you may need to employ previous SQL commands such as for joining tables, and so on. 

You can use Oracle Live SQL for both stages.

  • When you log in, go to My Scripts and select the JLDB_HOPE script that was previously used.
  • Re-run it so that the tables can be regenerated.
    • Wait about 30 seconds and then click on Schema to check the tables are all present.
    • If not, wait a few more seconds, then click Schema again.
  • Once all tables are present, click on SQL Worksheet and begin coding.
    • If you want to continue with the same database schema following the activities in "HOPE Session 1", you can go to My Session Previous Sessions, and rerun the actual session.
  • If your session gets ‘timed out’, just refresh the page, and repeat the process for re-running the JLDB_HOPE script. 

Please allow 40 to 80 minutes to complete both stages before you move on to the solutions below.

You will not need to upload your attempts. However, you can choose to save your SQL queries in a separate text file or as screenshots if you want to keep a record of your attempts.

 

Stage 1: Row Functions

The requests from the client are thus:

  • S1-1. List all of the book titles in mixed case (upper case for the first letters, the rest in lower case) along with the publisher’s name and the age of the book in months.
  • S1-2. Display the day, hour, minutes and seconds of the current date.
  • S1-3. Find out how many books each author wrote. Display author’s first and last name, their author id, and how many books (s)he wrote.
    • Sort by last name and then first name.
  • S1-4. List customers from Florida and Georgia who have placed an order totaling over $50.
    • Note that some customers may have placed more than one order.

 

Stage 2: Indexes, Constraints and Views

The requests from the client are thus:

  • S2-1. Add an index for the customer’s last name.
  • S2-2. Create a unique index on Title in the Books table.
  • S2-3. Create a view from the Books table in which a constraint ensures that all retail prices have to be between $25 and $55.
    • Call the view vBook.
