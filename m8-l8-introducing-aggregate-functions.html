 SELECT Statements Containing Aggregate Functions

 

There are typically five aggregate functions that are used within a SELECT statement.

Be sure to click on each tab to learn more about the aggregate functions.

COUNT

  • This returns the number of values in a specified column.
  • Can be applied to both numeric and non-numeric fields.

SUM

  • This returns the sum of the values in a specified column.
  • Can only be applied to numeric fields.

AVG

  • This returns the average (i.e. arithmetic mean) of the values in a specified column.
  • Can only be applied to numeric fields.

MIN

  • This returns the smallest (i.e. lowest) value in a specified column.
  • Can be applied to both numeric and non-numeric fields.

MAX

  • This returns the largest (i.e. highest) value in a specified column.
  • Can be applied to both numeric and non-numeric fields.

Each of these aggregate functions operates on a single column of a table and will return a single value.

With the exception of COUNT(*), each function will eliminate any NULL values first and will then operate on any non-NULL values that remain.

The COUNT(*) function will count all of the rows of a table irrespective of the presence of NULL values or duplicates.

It is possible to use DISTINCT before the column name in order to eliminate any duplicates. However, the DISTINCT keyword will not have any effect with MIN and MAX, but it may have an effect on SUM and AVG.

In addition to the SELECT keyword, aggregate functions can also be used in the HAVING clause (which is used alongside the GROUP BY clause). It will restrict the groups that appear in the final table. HAVING is similar to the WHERE clause but whereas the latter filters individual rows, the former filters the groups.
