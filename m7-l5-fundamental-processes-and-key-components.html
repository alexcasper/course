The Fundamental Processes and Services of a DBMS

We will now take a look at the fundamental processes pertaining to a DBMS, starting with the System Catalogue.

The integrated system catalogue is a fundamental process and thus a major component of a DBMS. It is also a key feature of the ANSI-SPARC architecture which can be accessible to both the registered users and the DBMS.

The system catalogue – also known as the ‘data dictionary’ – is a repository of information describing the data contained within the database.

In essence it is the metadata (“the data that describes the data”). The amount of information contained within, and how the information is used, depends on the DBMS in operation.

Typical Contents of the System Catalogue

The contents of the system catalogue can include:

  • The names, types and sizes of the data items.
  • The names of the relationships.
  • Integrity constraints on the data.
  • Names of authorized users who have access to the data.
  • The data items that each user can access and the types of access allowed:
    • Examples include insert, update, delete or read access.
  • The external, conceptual and internal levels (schemas), along with the mappings between the levels.
  • The usage statistics:
    • The frequencies of transactions.
    • The counts on the number of accesses made to the objects in the database.

Additional Benefits of the System Catalogue

Additional benefits of the system catalogue include:

  • The information concerning the data can be collected and stored centrally:
    • This ensures that control over the data as a resource is maintained.
  • The meaning of the data can be defined:
    • This assists other users to be able to understand the purpose of the data.
  • Simplification of the communication because the exact meanings are stored:
    • The system catalogue may be able to identify the users who own and/or access the data.
  • Redundancies and inconsistencies can be identified more easily as the data becomes centralized.
  • Alterations to the database can be recorded.
  • The impact of a change can be determined before it is implemented:
  • The system catalogue has recorded each data item, all of its relationships, and all of its users.
  • Security can be enforced.
  • Integrity can be ensured.
  • Audit information can be provided on request.

The system catalogue is one of ten services a DBMS must be able to provide.

Let's now take a look at the other key services (please click on the tabs to learn more):

1. Data storage, retrieval and updating

This is deemed the "raison d'être" for a DBMS, i.e. "the reason to be"!

2. Transaction support

The DBMS needs to ensure that a transaction will go to completion by applying all of the updates generated by a user, or is aborted thus no updates applied so as to avoid any inconsistencies in the database.

3. Concurrency control services

The DBMS must ensure that the database is updated correctly when there are multiple users making updates to the database at the same time.

This is very important when transactions involve money!

4. Recovery services

The DBMS must be able to restore the database to the previous consistent state following any damage.

5. Authorisation services

The DBMS must ensure that only authorised (i.e. registered) users can access the database, and that where necessary, some users are restricted to the information available to them.

6. Support for data communication

A DBMS must be able to integrate with communication software, especially if the user is located at an external site.

In this situation, a data communication manager (DCM) can be employed to enable transmissions from the DBMS to the user.

7. Integrity services

The DBMS ensures that the data that is contained in the database, and when changes are made to that data, all follow defined rules.

This ensures that the data is correct and consistent.

This can thus be maintained by applying a constraint.

8. Promotion of data independence

This is to ensure that facilities are included to enable the support of the independence of programs from the actual structure of the database.

On occasions, it can be difficult to attain logical data independence because of the difficulty in removing an entity, attribute or relationship, even though their corresponding addition is permissible.

9. Utility services

  • It is recommended that a DBMS should be able to provide a set of utility services in order to assist the DBA to administer the database effectively.
  • Some work at the external level; others at the internal level which can only be provided by the DBMS vendor. In terms of the internal level, these include:
    • Data import facilities
      • Loading the database from flat-files.
    • Data export facilities
      • Dumping the database into flat-files.
    • Monitoring facilities
      • To keep an eye on database usage and operations.
    • Statistical analysis programs
      • To keep an eye on performance and usage statistics.
    • Index reorganization facilities
    • Garbage collection and reallocation
      • This removes deleted records physically from the storage devices, consolidates the space released, and reallocates the space where it is required.

 What is concurrency control and why does a DBMS need a concurrency control facility?

Reveal Answer

This is a mechanism that ensures the database is correctly updated when multiple users are updating the database at the same time.

This avoids inconsistencies arising when two or more transactions are taking place at the same time in which one is updating the database.

 

 

The Key Components of a DBMS

The DBMS is partitioned into several software components (also known as modules), each of which is assigned a specific task.

It is worth noting that some of the functions of the DBMS are also supported by the underlying operating system.

The DBMS interfaces with other software components, such as user queries and access methods (involving file management techniques for storing and retrieving data records).

Which of the components would not necessarily be needed in a standalone PC DBMS?

Reveal Answer

Concurrency Control – Since there is only one single user.
