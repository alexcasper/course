What is a ServiceNow Instance?

The ServiceNow cloud is built on an advanced architecture called multi-instance. In this architecture, an organisation’s data, applications, and customisations reside in a unique software stack called an instance.

An organisation may have more than one instance e.g. Development, Test, Production etc and each instance is a single implementation of the ServiceNow platform. Each instance is isolated from the other instances, but they can still communicate with each other. The ServiceNow multi-instance architecture provides these distinct advantages:

An organisation’s data is truly isolated, making hardware and software maintenance on these unique customer instances easy to perform. Issues can be resolved on a customer-by-customer basis. The multi-instance architecture allows ServiceNow to perform actions, such as upgrading, on individual customer instances, on a schedule that fits the compliance requirements and needs of that customer.

You will be assigned your own NCH personal instance that you will retain for the duration of your programme.

 

Introducing ServiceNow Releases

ServiceNow deliver two major releases each year, typically in March and September and name their releases after global cities in alphabetical order. Each ServiceNow release contains enhancements to platform functionality. Each release comes with detailed release notes with comprehensive descriptions of new features and enhancements, functionality fixes, user interface changes, security improvements etc along with a planning checklist if you are looking to upgrade to a new release.
