Using CSV Files for the Database

Let's take a look at how CSV files for a database can be imported or exported. 

 

Importing CSV Files

This video will show you how to import a CSV text file into a database.




Transcript


When a CSV file is being imported into a database, it is very important to consider the following (please be sure to click on both tabs): 

The table does not currently exist inside the database.

  • After importing of the CSV file, the first row in that file will be interpreted as attribute (column) names. 
  • All other rows are the actual data.

The table already exists inside the database. 

  • Every row in the CSV file is considered to be the data. This means that the attribute names in the first row of the CSV file will be seen as data and inserted into the existing table as such.
  • After importing the new CSV file into an existing table, the first row pertaining to the new CSV file can be deleted from the updated table via a SQL query.
    • The deletion should only occur if the newly imported CSV file contained column headers. If it did not contain column headers then no deletion would be required.

Two useful SQLite3 dot commands are .tables and .schema:

  • .tables lists the names of the tables that are inside the current database.
  • .schema shows the attributes inside the tables along with their data types (or ‘domains’).
