 Numeric Functions

The numeric functions permit the manipulation of numerical data values.

Let's take a look at some examples (please be sure to click on all of the tabs below):

ROUND

  • Rounding a numerical value to the required precision (e.g. integer, decimal place).
  • For example:
    • ROUND(34.7)  35
    • ROUND(34.7, 0)  35
    • ROUND(34.7, 1)  34.7
    • ROUND(34.7, -1)  30
    • ROUND(38.7, -1)  40

TRUNC

  • Truncating sets the number of decimal places to allow after the decimal point.
  • For example:
    • TRUNC(65.873,1)  65.9
    • TRUNC(65.873,0) 66
    • TRUNC(65.873,2) 65.87

POWER

  • This takes a number and raises it to a required power value.
  • For example:
    • POWER(4,3)  64 (i.e. 4×4×4 = 64)

MOD

  • The modulus gives the “remainder” following a division.
  • For example:
    • MOD(9,7)  7 goes into 9 once with a remainder of 2  modulus output of 2).

ABS

  • The absolute value converts any negative numerical value into a positive value.
    • ABS(-24.6) 24.6.

The ROUND Function

Click on the tab below to reveal an illustration of the effects of precision on using the ROUND() function in Oracle Live SQL.

Example outputs employing the ROUND() Function

When you study the output from the ROUND() function, you should observe:

  • If the precision is 0, it rounds to the nearest integer, else if 1 then to the first decimal place, else if 2 then to the second decimal place.
  • If precision is -1 then it rounds the unit to 0 and the tens to the nearest integer depending on the value of the unit, so
    • if we have 26, the 6 is changed to a zero, and the 2 is rounded up to a 3.
    • If we have 24, then the 4 is changed to a zero, and the ten remains at 2.
  • If the precision is -2, then it rounds the units and tens to zero and rounds the hundreds to the nearest integer depending on the values of the tens.

In SQLite3, the ROUND function appears to only round to the nearest integer (but displays to one decimal place, e.g. 14.0); setting the precision appears to have no effect on the output, in contrast to what was shown in the PDF.

Additional Date Functions

Click on the tabs below to view more examples of date functions that can be employed in Oracle Live SQL.

 

ADD_MONTHS

  • Add a specified number of months to a given date.
  • For example:
    • If the current date is Monday 8th June, and you want to add 4 months to it, then the ADD_MONTHS() function can be used as follows: ADD_MONTHS(08-06-20,4).
      • The output should be 08-10-20 (8th October).

NEXT_DAY

  • This function works out what the next date will be given a specific day of the week.
  • For example:
    • If the current date is Monday 8th June, and you want to know the date of the Thursday coming, you can use the NEXT_DAY() function as follows: NEXT_DAY(current_date, ‘THURSDAY’).
      • The output should be 11th  June.

TO_DATE

  • This converts various date formats to the internal date format (DD-MON-YY).
  • For example:
    • To convert the ‘June 30, 2008’ to the internal date format, you would say TO_DATE(‘March 31, 2008’, ‘Month DD, YYYY’).
      • The output should be 30-MAR-08.

LAST_DAY

  • Gives the last numerical day of the month.
  • For example:
    • If the month is May then LAST_DAY gives 31.

SYSDATE and CURRENT_DATE

Give the current date according to the system hosting the database, and the current date based on the “locale” respectively.
